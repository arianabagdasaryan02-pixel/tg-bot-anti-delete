# Как запустить бота

Разбор всех способов развернуть бота — от локального теста до продакшена на VPS.

> ⚠️ **Важная особенность Telegram Business API:** если бот выключен в момент удаления или редактирования сообщения — событие **теряется**. Telegram не хранит очередь недоставленных обновлений. Поэтому для полноценной работы нужен постоянно запущенный бот (24/7).

## Вариант 1: Локально на своём компьютере

Подходит для: **тестирования, разработки, знакомства с кодом**.

### Что понадобится

- Python 3.10 или новее
- Стабильный интернет
- Установленный `git` (опционально, можно скачать код архивом)

### Windows

**1. Установи Python**

Скачай с [python.org](https://www.python.org/downloads/). При установке **обязательно** поставь галочку «Add Python to PATH».

Проверь установку:
```powershell
python --version
```

**2. Скачай код**

Через git:
```powershell
git clone https://github.com/<username>/<repo>.git
cd <repo>
```

Или скачай ZIP с GitHub → «Code» → «Download ZIP» → распакуй.

**3. Создай виртуальное окружение**

```powershell
python -m venv venv
venv\Scripts\activate
```

После активации в начале строки терминала появится `(venv)`.

**4. Установи зависимости**

```powershell
pip install -r requirements.txt
```

**5. Настрой конфиг**

```powershell
copy config.example.py config.py
notepad config.py
```

Впиши свой токен от [@BotFather](https://t.me/BotFather) и имя бота.

**6. Запусти**

```powershell
python bot.py
```

Всё, бот работает. Закроешь окно — остановится.

### macOS / Linux

**1. Python обычно уже установлен.** Проверь:
```bash
python3 --version
```

Если нет — на macOS через [Homebrew](https://brew.sh/): `brew install python`. На Ubuntu/Debian: `sudo apt install python3 python3-venv python3-pip`.

**2. Скачай код и настрой:**

```bash
git clone https://github.com/<username>/<repo>.git
cd <repo>

python3 -m venv venv
source venv/bin/activate

pip install -r requirements.txt

cp config.example.py config.py
nano config.py  # или любой другой редактор
```

**3. Запусти:**

```bash
python bot.py
```

### Как не выключать бота на своём компе

Проблема: бот работает пока открыт терминал. Решения:

**macOS / Linux — через `nohup`:**
```bash
nohup python bot.py > bot.log 2>&1 &
```
Бот продолжит работать после закрытия терминала. Остановить: `pkill -f bot.py`.

**macOS / Linux — через `screen` или `tmux`:**
```bash
screen -S bot
python bot.py
# Ctrl+A, потом D — отсоединиться, бот продолжит работать
# screen -r bot — вернуться в сессию
```

**Windows — через Планировщик заданий:** создай задачу с триггером «при входе в систему», действие — `python bot.py`.

### Ограничения локального запуска

- ❌ Бот работает только пока компьютер включён и есть интернет
- ❌ Закрыл ноутбук / отключился интернет → пропустил все удаления и правки
- ❌ Компьютер шумит и потребляет электричество 24/7
- ✅ Полный контроль, легко отлаживать, ничего не платишь

## Вариант 2: VPS (рекомендуется)

Подходит для: **постоянной работы 24/7**.

VPS — это виртуальный сервер в дата-центре, к которому ты подключаешься по SSH. Работает круглосуточно без твоего участия.

### Какой VPS выбрать

Для этого бота нужен минимум: **1 CPU, 512 MB RAM, 10 GB диска, Ubuntu 22.04 или новее**. Практически любой самый дешёвый тариф подойдёт.

**Российские провайдеры** (оплата картой РФ):
- [Beget](https://beget.com/) — от ~200₽/мес, 30 дней бесплатного тестового периода
- [Timeweb Cloud](https://timeweb.cloud/) — от ~200₽/мес
- [Selectel](https://selectel.ru/) — от ~250₽/мес, есть Cloud-серверы с почасовой оплатой
- [SmartApe](https://smartape.ru/) — 10 дней бесплатного VPS без ввода карты
- [Aeza](https://aeza.net/), [RUVDS](https://ruvds.com/) — от ~150₽/мес

**Зарубежные** (нужна иностранная карта):
- [Hetzner](https://www.hetzner.com/) — от €4/мес, отличное соотношение цена/качество
- [Contabo](https://contabo.com/) — от €4/мес, много ресурсов за деньги
- [DigitalOcean](https://www.digitalocean.com/) — от $4/мес, часто есть стартовые кредиты

### Пошаговая настройка на Ubuntu

Предполагаем, что ты уже заказал сервер и знаешь его IP-адрес, логин (обычно `root`) и пароль.

**1. Подключись по SSH**

С macOS/Linux:
```bash
ssh root@ВАШ_IP
```

С Windows: используй встроенный SSH в PowerShell (Windows 10+) или [PuTTY](https://www.putty.org/).

**2. Обнови систему**

```bash
apt update && apt upgrade -y
```

**3. Установи Python и git**

```bash
apt install -y python3 python3-venv python3-pip git
```

**4. Создай отдельного пользователя для бота** (не запускай под `root`!)

```bash
adduser botuser
# задай пароль, остальное можно оставить пустым (Enter)

# переключись на него
su - botuser
```

**5. Скачай код и настрой**

```bash
cd ~
git clone https://github.com/<username>/<repo>.git bot
cd bot

python3 -m venv venv
source venv/bin/activate

pip install -r requirements.txt

cp config.example.py config.py
nano config.py
# впиши BOT_TOKEN и BOT_USERNAME, сохрани (Ctrl+O, Enter, Ctrl+X)
```

**6. Проверь, что запускается**

```bash
python bot.py
```

Если пишет «Бот запущен» — всё ок. Останавливай `Ctrl+C`.

**7. Настрой автозапуск через systemd**

Вернись под root:
```bash
exit  # выходим из botuser обратно в root
```

Создай сервис:
```bash
nano /etc/systemd/system/tgbot.service
```

Вставь:
```ini
[Unit]
Description=Telegram Business Bot
After=network.target

[Service]
Type=simple
User=botuser
WorkingDirectory=/home/botuser/bot
ExecStart=/home/botuser/bot/venv/bin/python bot.py
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

Сохрани (Ctrl+O, Enter, Ctrl+X). Запусти:

```bash
systemctl daemon-reload
systemctl enable tgbot
systemctl start tgbot
```

Проверь:
```bash
systemctl status tgbot
```

Должно быть `active (running)`. Теперь бот перезапустится сам при падении и при перезагрузке сервера.

Логи смотреть так:
```bash
journalctl -u tgbot -f
```

**8. Настрой автоочистку через cron**

```bash
su - botuser
crontab -e
```

Добавь строку (каждый день в 03:00):
```
0 3 * * * cd /home/botuser/bot && /home/botuser/bot/venv/bin/python cleanup.py >> cleanup.log 2>&1
```

Сохрани и выйди.

**Готово.** Бот работает 24/7, перезапускается при сбоях, каждую ночь чистит старые данные.

### Как обновлять код на сервере

```bash
su - botuser
cd ~/bot
git pull
# если менялись зависимости:
source venv/bin/activate
pip install -r requirements.txt
exit
# перезапуск сервиса под root:
systemctl restart tgbot
```

## Вариант 3: Бесплатные хостинги для ботов

Подходит для: **тех, кто не хочет платить и разбираться с Linux**.

Важно: **полностью бесплатных хостингов без ограничений почти не существует.** Обычно это либо бесплатные тарифы с лимитами, либо пробные периоды.

### Bothost (bothost.ru)

Российский специализированный хостинг для ботов. Есть бесплатный тариф: 1 бот, 256 MB RAM, без банковской карты. Разворачивание через веб-интерфейс за несколько минут.

Плюсы: интерфейс на русском, поддержка Python из коробки, автодеплой из GitHub.
Минусы: 256 MB RAM могут быть на грани для хранения медиа-архива.

### Amvera Cloud (amvera.ru)

Российский PaaS, часто называют «российским Heroku». Есть небольшой начальный баланс, потом ~150-300₽/мес.

Плюсы: деплой через git push, автомасштабирование, есть логи в веб-панели.
Минусы: полностью бесплатным не назвать — только стартовый бонус.

### Railway (railway.app)

Западный сервис. Бесплатно даёт $5 кредита в месяц — на маленького бота хватает.

Плюсы: удобный веб-интерфейс, автодеплой из GitHub.
Минусы: нужна иностранная карта для верификации, интерфейс на английском.

### Render (render.com)

Западный сервис с бесплатным тарифом. Основная проблема: сервисы **«засыпают» после 15 минут неактивности**, что критично для нашего бота (пока «спит» — пропускает удаления).

Для этого бота Render **не подходит** без платного тарифа.

### PythonAnywhere (pythonanywhere.com)

Специализированный хостинг для Python. Бесплатный тариф имеет ограничения на исходящие HTTP-запросы (белый список доменов) — Telegram API там есть, но `getFile` для больших медиа может не работать.

### Общее правило для бесплатных хостингов

- Проверь, не «засыпает» ли сервис при неактивности — для Business API это критично
- Учти лимит по диску: у бота растёт `media_archive/` и `messages.db`
- Настрой `cleanup.py` **обязательно** — на бесплатных тарифах диск маленький

## Вариант 4: Домашний мини-сервер

Подходит для: **тех, кто любит железо и не хочет платить ежемесячно**.

### Raspberry Pi

Одноплатный компьютер размером с колоду карт. Модели от 4 000₽ (Pi Zero 2 W) до 12 000₽ (Pi 5). Потребляет ~3-7 Вт (копейки за электричество).

Установка: Raspberry Pi OS (Linux) → дальше всё как в разделе про VPS.

Плюсы: один раз купил — работает вечно, данные дома, не платишь за хостинг.
Минусы: зависит от твоего интернета и электричества, при отключении света бот падает.

### Старый ноутбук / компьютер

Если валяется ненужный ноут с рабочим экраном — можно поставить туда Ubuntu Server или Debian и использовать как сервер.

Плюсы: бесплатно, много ресурсов.
Минусы: шум, потребление электричества (у ноутбука ~20-40 Вт), греется.

Совет: закрой крышку ноутбука, но не давай ему уходить в сон. В Ubuntu:
```bash
sudo nano /etc/systemd/logind.conf
# найди HandleLidSwitch=suspend, замени на HandleLidSwitch=ignore
sudo systemctl restart systemd-logind
```

### Динамический IP и роутер

**Хорошая новость:** для этого бота **не нужно** настраивать проброс портов, белый IP или динамический DNS. Бот работает через **polling** (сам стучится в Telegram), а не через webhook. Достаточно того, что у тебя есть исходящий интернет.

## Сравнение вариантов

| Вариант | Цена | Работает 24/7 | Сложность | Для кого |
|---------|------|---------------|-----------|----------|
| Свой компьютер | 0₽ | ❌ Только при включённом ПК | 🟢 Легко | Тест, разработка |
| VPS | 150-500₽/мес | ✅ Да | 🟡 Средне | Постоянная работа |
| Bothost / Amvera | 0-300₽/мес | ✅ Да (проверь SLA) | 🟢 Легко | Новички без Linux |
| Railway / Render | $0-5/мес | ⚠️ Могут «засыпать» | 🟢 Легко | Не подходит для Business API |
| Raspberry Pi | ~5 000₽ разово | ✅ Пока есть свет | 🟡 Средне | Энтузиасты |
| Старый ноут | 0₽ | ✅ Пока не сломается | 🟡 Средне | Есть лишнее железо |

## Что рекомендую

- **Хочу попробовать и понять, как работает** → локально на своём компе
- **Нужна постоянная работа, готов разобраться** → VPS (Beget, Timeweb, Hetzner)
- **Не хочу возиться с Linux** → Bothost или Amvera
- **Люблю железо, есть Raspberry Pi** → домашний сервер
